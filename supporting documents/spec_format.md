**文档整体结构**

1. 系统总览（架构图 + 核心原则 + 数据流向）
2. 客户文件结构（文件清单 + 各文件一句话定位）
3. 数据文件规格（每个文件一个章节，用统一模板）
4. 主 Workflow 节点（每个节点一个章节，用统一模板）
5. Agent 规格（每个 Agent 一个章节，用统一模板）
6. 维护流程
7. 最终输出格式
8. 待设计 / 待定事项


**三套标准模板：**

**数据文件模板**（适用于 rules.md / observations.md / profile.md / coa.csv / intervention_log.md 等）

1. 职责定义——这个文件存在的唯一目的是什么
2. 数据格式——字段定义、示例
3. 数据来源——有哪些写入方式、谁可以写入
4. 状态与生命周期——有哪些状态、状态之间如何流转、清理/过期条件
5. 被谁读取——哪些节点 / Agent 在什么时候读取这个文件
6. 被谁修改——哪些节点 / Agent 有权修改、修改的触发条件
7. 与其他组件的关系——和其他文件 / 节点 / Agent 的上下游依赖

**Workflow 节点模板**（适用于 Node 1 / Node 2 / Node 3 / JE 构造与校验 / 输出报告等）

1. 职责定义——这个节点做什么
2. 输入——接收什么数据、从哪来
3. 执行逻辑——具体怎么处理、分支条件是什么
4. 输出——产出什么、输出到哪去
5. 异常处理——匹配失败 / 数据缺失时怎么办
6. 与其他组件的关系——读取哪些文件、写入哪些文件、触发哪些下游

**Agent 模板**（适用于 Onboarding / Coordinator / Review Agent）

1. 职责定义——这个 Agent 的唯一职责是什么、边界在哪里
2. 触发条件——什么时候启动、由谁触发
3. 输入——接收什么数据、从哪来
4. 执行流程——分步骤描述，包含分支条件和决策依据
5. 输出——产出什么文件或操作结果
6. 权限边界——允许做什么、明确禁止做什么
7. 与人的交互——和谁沟通、沟通什么内容、沟通形式
8. 与其他组件的关系——上游输入来源、下游触发对象、不直接交互的组件

**共享工具模板**（适用于 tools/ 目录下的共享工具，如 standardize_description）

1. 职责定义——这个工具解决什么问题、为什么作为共享工具而非节点
2. 接口——函数签名、输入参数、返回字段及各字段的下游用途
3. 内部实现——处理管线或算法逻辑，包含分支条件和降级策略
4. 内部存储——工具维护的持久化数据（格式、schema、存储位置、生命周期）
5. 错误处理——异常场景的降级策略、标记方式、后续修正路径
6. 被谁调用——哪些节点 / Agent 在什么时机调用、调用时的操作类型
7. 文件位置——实现文件的目录结构
